# TroubleShoot.md - Render 139 에러 (Segmentation Fault)

## 이슈 개요
- **현상**: Render Free 인스턴스에서 서비스 운영 중 `exit status 139`와 함께 서버가 반복적으로 재시작됨.
- **발생 시점**: 주로 서버 시작(Startup) 및 Daily Batch Job 실행 시점.
- **로그 증상**: `Error fetching KRX data: No module named 'pykrx'` 로그 직후 또는 `update_history_job` 실행 중 크래시 추정.

## 원인 분석 (가설)
1. **메모리 부족 (OOM)**: Render Free 인스턴스는 512MB RAM을 제공함. Startup 시 `yfinance`와 `pandas`를 사용하여 10여 개 지표의 1년치 역사적 데이터를 한꺼번에 로드하면서 메모리 임계치를 초과, 139(Segmentation Fault) 에러 유발.
2. **의존성 결여**: `requirements.txt`에 `pykrx`가 누락되어 `ImportError`가 발생하고 있으나, 이는 139 에러의 직접적 원인이라기보다 시스템 불안정 요인으로 작용.
3. **네이티브 라이브러리 충돌**: `yfinance`나 `pandas`의 C-extension이 특정 데이터 처리 중 메모리 접근 오류를 일으켰을 가능성.

### [Stock] 하이일드 스프레드 및 기준금리 데이터 누락
- **원인**: IndexerGo 및 Investing.com의 빈번한 403 에러 및 크롤링 차단.
- **분석**: Render 등 클라우드 IP에 대한 웹사이트 측의 보안 강화로 인해 단순 헤더 보완만으로는 한계가 있음.
- **조치**: 
    1. **하이일드 스프레드**: **IndexerGo 크롤링을 주력**으로 복구하되, 실패 시 FRED API(`BAMLH0A0HYM2`)로 폴백하는 이중 구조 적용.
    2. **미국 기준금리**: **Investing.com 크롤링을 주력**으로 복구하여 상세 일정 정보를 유지하며, 수집 실패 시 FRED API(`FEDFUNDS`)로 폴백하여 데이터 연속성 보장.

### [Exchange] 외국인 주식 보유 데이터 누락 (KRX)
- **원인**: KRX 정보데이터시스템의 보안 강화로 인한 로그인 필수(회원제) 전환 및 보안 프로그램 설치 요구. 
- **분석**: 서버 환경(Render)에서는 보안 프로그램 설치가 불가능하며, 계정 로그인 방식은 캡차 및 계정 잠김 위험이 있어 지속 가능성이 낮음.
- **조치**: 
    1. 수집원을 **e-나라지표(index.go.kr)**의 '외국인 증권투자 현황' 공식 통계로 변경.
    2. **프론트엔드 연동**: 대시보드 내 '외국인 주식 보유' 클릭 시 이동하는 URL을 [e-나라지표 상세 페이지](https://www.index.go.kr/unity/potal/main/EachDtlPageDetail.do?idx_cd=1086)로 업데이트 완료.
    3. **특징**: 일간 데이터 대신 **월간 공식 데이터**를 기반으로 하여 수집 안정성을 최우선으로 확보함. (보안 프로그램이나 로그인 불필요)

## 이슈 3: Render 배포 지연 ("Deploying..." 상태 지속)
- **현상**: 빌드 성공 후 배포 단계에서 상태 업데이트가 지연됨.
- **원인 분석**: Render Free 인스턴스는 배포 시 이전 인스턴스와 새로운 인스턴스의 교체 과정에서 헬스체크(`/health`) 성공을 기다림. 서버 시작 시 실행되는 `run_startup_jobs`가 비동기로 동작하지만, 초기 네트워크 부하 등으로 인해 헬스체크 응답이 지연될 수 있음.
- **조치 사항**: 부하 분산(sleep 추가)을 적용하였으므로, 배포 시 약 1~2분 정도 충분히 대기하면 정상적으로 Live 상태로 전환됨.

## 최종 조치 결과 (요약)
1. **메모리 최적화 (139 에러)**: Startup Job 딜레이 및 History 수집 순차화 완료. (Segment Fault 해결)
2. **의존성 해결**: `pykrx`, `setuptools` 추가 완료.
3. **데이터 수집 안정화 (하이브리드 모델)**: 
    - **하이일드 스프레드**: FRED API(`BAMLH0A0HYM2`)로 전환하여 403 에러 원천 차단.
    - **미국 기준금리**: **FRED(실제수치) + Investing.com(일정)** 하이브리드 수집 로직 적용.
    - **외국인 주식 보유**: **e-나라지표(월간 공식 데이터)**로 전환하여 KRX 로그인/보안 프로그램 제약 우회.
4. **결과**: 주요 지표 데이터 누락 해결 및 인프라 안정성 확보 완료.
